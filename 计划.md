# SmartRecruit 智能简历推荐系统 — 从零复现完整计划

## 上下文

此计划面向一个基于 RAG 架构的智能招聘助手系统。核心能力：多格式简历入库 → AI 解析结构化信息 → 用户自然语言招聘需求 → 混合检索匹配候选人 → 多轮追问交互。前后端一体化（Streamlit），底层整合 Milvus + Elasticsearch + MongoDB 三大存储引擎。

**目标**：让一个具备 Python 基础的开发者能够按照本计划，从空文件夹开始，一步步复现整个项目。每个阶段包含：要创建的文件、要写的代码、为什么要这么写、写完后的验证方法。

## Phase 3：向量存储层 — `utils/vector_store.py`（约 3 小时）

### 目标
这是整个系统最底层的核心模块。实现：Milvus 集合创建、简历存储（同时写三个库）、混合检索 + 重排序。

**这一步是整个项目最关键也最复杂的一步**，因为后续所有检索功能都依赖它。

### Step 3.1 — 理解要做什么

`VectorStore` 类需要管理三个存储后端：

| 后端 | 存什么 | 用途 |
|------|--------|------|
| Milvus | 稠密向量(1024d) + 稀疏向量 + 标量字段(gender/age/experience) | 语义+关键词混合搜索 |
| Elasticsearch | 每个 chunk 的文本 + 元数据 | BM25 关键词全文检索 |
| MongoDB | 简历完整文本 + 元数据 + 结构化数据 | 元数据查询、去重、完整简历获取 |

整个类的方法清单：
- `_initialize_components()` — 连接 Milvus/ES/MongoDB，加载 BGE-M3 和 Reranker 模型
- `_create_or_load_collection()` — 创建 Milvus 集合的 schema 和索引
- `store_resume(doc, chunks, structured_data)` — 一份简历切块后同时写三库
- `get_metadata_by_hash(doc_hash)` — MongoDB 按 hash 查元数据
- `get_full_resume(doc_hash)` — MongoDB 按 hash 取完整文本
- `hybrid_search_with_rerank(query, params)` — 混合检索 + 重排核心方法
- `aget_relevant_documents(query, params)` — 异步包装器

**② 类定义和 `_initialize_components()`：**

```python
class VectorStore:
    def __init__(self):
        self._initialize_components()

    def _initialize_components(self):
        logger.info("开始初始化向量存储及检索组件...")
        try:
            # 1. 连接 Milvus
            self.client = MilvusClient(uri=f"http://{config.MILVUS_HOST}:{config.MILVUS_PORT}")

            # 2. 加载 BGE-M3 嵌入模型
            embedding_model_path = os.path.join(config.MODEL_PATH, config.EMBEDDING_MODEL)
            self.embedding_function = BGEM3EmbeddingFunction(
                model_name=embedding_model_path,
                device='cpu',
                use_fp16=False
            )
            self.dense_dim = self.embedding_function.dim["dense"]  # 1024

            # 3. 加载重排模型
            reranker_model_path = os.path.join(config.MODEL_PATH, config.RERANKER_MODEL)
            self.reranker = CrossEncoder(reranker_model_path)

            # 4. 连接 MongoDB
            mongo_uri = (
                f"mongodb://{config.MONGO_USER}:{config.MONGO_PASSWORD}"
                f"@{config.MONGO_HOST}:{config.MONGO_PORT}/{config.MONGO_DB}?authSource=admin"
            )
            self.mongo_client = MongoClient(mongo_uri, serverSelectionTimeoutMS=5000)
            self.mongo_client.admin.command("ping")  # 测试连接
            self.mongo_db = self.mongo_client[config.MONGO_DB]
            self.mongo_collection = self.mongo_db["resumes"]

            # 5. 连接 Elasticsearch
            self.es_client = Elasticsearch(config.ES_HOST)
            if not self.es_client.indices.exists(index=config.ES_INDEX_NAME):
                self.es_client.indices.create(index=config.ES_INDEX_NAME)

            # 6. 创建/加载 Milvus 集合
            self._create_or_load_collection()

            logger.info("向量存储及检索组件初始化完成")
        except Exception as e:
            logger.critical(f"组件初始化失败: {e}", exc_info=True)
            raise
```

**③ `_create_or_load_collection()` — Milvus Schema 设计：**

```python
    def _create_or_load_collection(self):
        collection_name = config.MILVUS_COLLECTION_NAME
        if not self.client.has_collection(collection_name):
            schema = MilvusClient.create_schema(
                auto_id=False,
                enable_dynamic_field=True
            )
            # 主键
            schema.add_field("id", DataType.VARCHAR, is_primary=True, max_length=100)
            # 稠密向量 (语义匹配)
            schema.add_field("dense_vector", DataType.FLOAT_VECTOR, dim=self.dense_dim)
            # 稀疏向量 (关键词匹配)
            schema.add_field("sparse_vector", DataType.SPARSE_FLOAT_VECTOR)
            # 元数据字段
            schema.add_field("doc_hash", DataType.VARCHAR, max_length=32)
            schema.add_field("text", DataType.VARCHAR, max_length=65535)
            schema.add_field("gender", DataType.VARCHAR, max_length=10)
            schema.add_field("age", DataType.INT64)
            schema.add_field("work_experience", DataType.INT64)

            # 索引
            index_params = self.client.prepare_index_params()
            index_params.add_index(
                field_name="dense_vector", index_name="dense_index",
                index_type="IVF_FLAT", metric_type="IP",
                params={"nlist": 128}
            )
            index_params.add_index(
                field_name="sparse_vector", index_name="sparse_index",
                index_type="SPARSE_INVERTED_INDEX", metric_type="IP",
                params={"drop_ratio_build": 0.2}
            )

            self.client.create_collection(
                collection_name=collection_name,
                schema=schema,
                index_params=index_params
            )
        self.client.load_collection(collection_name)
```

> **设计要点**：
> - `IVF_FLAT` 索引适合中等规模（<100万向量），IP（内积）度量适用于归一化后的向量
> - `SPARSE_INVERTED_INDEX` 是 Milvus 对稀疏向量的原生索引类型
> - 标量字段 `gender`/`age`/`work_experience` 支持在检索时做 SQL-like 过滤
> - `enable_dynamic_field=True` 允许后续灵活添加字段

**④ `store_resume()` — 三库同时写入：**

```python
    def store_resume(self, doc: Document, chunks: List[Document],
                     structured_data: Dict[str, Any]) -> bool:
        doc_hash = doc.metadata["hash"]
        # 去重检查
        if self.mongo_collection.find_one({"doc_hash": doc_hash}):
            return False
        if not chunks:
            return False

        # 生成所有 chunk 的嵌入向量
        texts = [chunk.page_content for chunk in chunks]
        embeddings = self.embedding_function.encode_documents(texts)

        # 构造 Milvus 插入数据
        data_to_insert = []
        for idx, chunk in enumerate(chunks):
            # 从 CSR 格式提取稀疏向量
            sparse_indices = embeddings["sparse"].indices[
                embeddings["sparse"].indptr[idx]:embeddings["sparse"].indptr[idx+1]
            ]
            sparse_data = embeddings["sparse"].data[
                embeddings["sparse"].indptr[idx]:embeddings["sparse"].indptr[idx+1]
            ]
            sparse_vector = {int(k): float(v) for k, v in zip(sparse_indices, sparse_data)}

            chunk_data = {
                "id": chunk.metadata["id"],
                "text": chunk.page_content,
                "dense_vector": embeddings["dense"][idx].tolist(),
                "sparse_vector": sparse_vector,
                "doc_hash": doc_hash,
                **structured_data,
                "parent_content": chunk.metadata.get("parent_content", "")
            }
            data_to_insert.append(chunk_data)

        try:
            # 写入 Milvus
            self.client.insert(config.MILVUS_COLLECTION_NAME, data_to_insert)
            # 写入 Elasticsearch
            for chunk in chunks:
                es_doc = {
                    "content": chunk.page_content,
                    "metadata": chunk.metadata
                }
                self.es_client.index(
                    index=config.ES_INDEX_NAME,
                    id=chunk.metadata["id"],
                    document=es_doc
                )
            # 写入 MongoDB（存完整文本）
            mongo_doc = {
                "doc_hash": doc_hash,
                "content": doc.page_content,
                "metadata": doc.metadata,
                "structured_data": structured_data
            }
            self.mongo_collection.insert_one(mongo_doc)
            logger.info(f"成功存储简历, hash: {doc_hash}")
            return True
        except Exception as e:
            logger.error(f"存储简历失败: {e}", exc_info=True)
            raise
```

**⑤ `hybrid_search_with_rerank()` — 核心检索逻辑（最重要！）：**

```python
    def hybrid_search_with_rerank(self, query: str, params: Dict[str, Any]) -> List[Document]:
        # 检索数量
        k = params.get('count', 1) * 1  # 初始召回数
        m = params.get('count', 1)      # 最终返回数

        # 构造过滤条件（Milvus 标量过滤）
        filter_conditions = []
        if params.get('gender') and params['gender'] != '未提供':
            filter_conditions.append(f"gender == '{params['gender']}'")
        if params.get('age_min') is not None:
            filter_conditions.append(f"age >= {params['age_min']}")
        if params.get('age_max') is not None:
            filter_conditions.append(f"age <= {params['age_max']}")
        if params.get('experience_min') is not None:
            filter_conditions.append(f"work_experience >= {params['experience_min']}")
        if params.get('experience_max') is not None:
            filter_conditions.append(f"work_experience <= {params['experience_max']}")
        filter_expr = " and ".join(filter_conditions)

        logger.info(f"开始混合检索: query='{query}', m={m}, filter='{filter_expr}'")

        # Step A: 查询编码
        query_embeddings = self.embedding_function.encode_queries([query])
        dense_vector = query_embeddings["dense"][0].tolist()
        sparse_vector = {
            int(idx): float(val)
            for idx, val in zip(
                query_embeddings["sparse"].indices,
                query_embeddings["sparse"].data
            )
        }

        # Step B: Milvus 混合搜索（稠密 70% + 稀疏 30%）
        dense_req = AnnSearchRequest(
            data=[dense_vector], anns_field="dense_vector",
            param={"metric_type": "IP", "params": {"nprobe": 10}}, limit=k
        )
        sparse_req = AnnSearchRequest(
            data=[sparse_vector], anns_field="sparse_vector",
            param={"metric_type": "IP"}, limit=k
        )
        milvus_results = self.client.hybrid_search(
            collection_name=config.MILVUS_COLLECTION_NAME,
            reqs=[dense_req, sparse_req],
            ranker=WeightedRanker(0.7, 0.3),
            limit=k, filter=filter_expr,
            output_fields=['id', 'work_experience', 'age', 'name', 'doc_hash',
                           'text', 'parent_content', 'gender']
        )
        milvus_results = milvus_results[0]
        logger.info(f"Milvus 召回 {len(milvus_results)} 个结果")

        # Step C: ES BM25 全文检索（补充召回）
        es_results = self.es_client.search(
            index=config.ES_INDEX_NAME,
            body={"query": {"match": {"content": query}}, "size": k}
        )["hits"]["hits"]
        logger.info(f"ES 召回 {len(es_results)} 个结果")

        # Step D: 合并去重
        all_hits = {hit['id']: hit['entity'] for hit in milvus_results}
        for hit in es_results:
            if hit['_id'] not in all_hits:
                mongo_data = self.get_metadata_by_hash(
                    hit['_source'].get('metadata', {}).get('hash')
                )
                if mongo_data:
                    all_hits[hit['_id']] = {
                        **hit['_source']['metadata'],
                        **mongo_data.get('structured_data', {})
                    }
        if not all_hits:
            return []

        # Step E: CrossEncoder 重排序
        pairs = [
            [query, entity.get('text', entity.get('parent_content', ''))]
            for entity in all_hits.values()
        ]
        scores = self.reranker.predict(pairs)

        docs_with_scores = [
            Document(
                page_content=entity.get('text', ''),
                metadata={**entity, 'rerank_score': score}
            )
            for entity, score in zip(all_hits.values(), scores)
        ]
        docs_with_scores.sort(key=lambda x: x.metadata['rerank_score'], reverse=True)

        # Step F: 按 doc_hash 去重，获取完整简历
        final_docs = []
        seen_hashes = set()
        for doc in docs_with_scores:
            doc_hash = doc.metadata.get('doc_hash')
            if doc_hash and doc_hash not in seen_hashes:
                mongo_doc = self.get_metadata_by_hash(doc_hash)
                if mongo_doc:
                    final_metadata = mongo_doc.get('metadata', {})
                    final_metadata['rerank_score'] = doc.metadata.get('rerank_score')
                    final_metadata['doc_hash'] = doc_hash
                    final_docs.append(Document(
                        page_content=mongo_doc.get('content', ''),
                        metadata=final_metadata
                    ))
                    seen_hashes.add(doc_hash)
            if len(final_docs) >= m:
                break

        logger.info(f"重排去重后返回 {len(final_docs)} 份独立简历")
        return final_docs
```

> **设计要点**：
> - `WeightedRanker(0.7, 0.3)` 表示稠密向量得分占 70%，稀疏向量占 30%
> - ES BM25 作为补充召回来源，弥补向量搜索可能漏掉的关键词精准匹配
> - CrossEncoder 重排比 Bi-Encoder 检索更精准，但速度更慢，所以只对 Top-K 候选做
> - 最终按 `doc_hash` 去重：同一份简历可能被多个 chunk 命中，只返回完整的一份

def compute_file_hash(file_path: str) -> str:
    """计算文件的 MD5 哈希，用于去重"""
    hasher = hashlib.md5()
    try:
        with open(file_path, "rb") as f:
            for chunk in iter(lambda: f.read(4096), b""):
                hasher.update(chunk)
        return hasher.hexdigest()
    except Exception as e:
        logger.error(f"计算文件hash失败: {file_path}, 错误: {str(e)}")
        raise

def load_and_hash_document(file_path: str, client: OpenAI) -> tuple:
    """
    加载文件内容 + 计算哈希
    返回: (文档文本内容, MD5哈希值)
    """
    logger.info(f"开始加载并哈希文件: {file_path}")
    file_extension = os.path.splitext(file_path)[1].lower()

    if file_extension not in document_loaders:
        raise ValueError(f"不支持的文件类型: {file_extension}")

    try:
        if file_extension in [".jpg", ".png"]:
            content = extract_text_from_image(file_path, client)
        else:
            loader_class = document_loaders[file_extension]
            if file_extension == ".txt":
                # TXT 文件尝试多种编码
                for enc in ["utf-8", "gbk", "latin1"]:
                    try:
                        loader = loader_class(file_path, encoding=enc)
                        content = loader.load()[0].page_content
                        break
                    except UnicodeDecodeError:
                        continue
                else:
                    raise UnicodeDecodeError(
                        f"无法以支持的编码加载文件: {file_path}", b"", 0, 0, ""
                    )
            else:
                loader = loader_class(file_path)
                content = loader.load()[0].page_content

        doc_hash = compute_file_hash(file_path)
        logger.info(f"文件加载并哈希成功: {file_path}, hash: {doc_hash}")
        return content, doc_hash
    except Exception as e:
        logger.error(f"加载或哈希文件失败: {file_path}, 错误: {str(e)}")
        raise

**⑥ 父子块切分策略：**

```python
def process_document(doc: Document) -> List[Document]:
    """
    父子块切分策略：
    1. 先将文档切成父块 (1000 chars, 100 overlap)
    2. 再将每个父块切成子块 (400 chars, 100 overlap)
    3. 每个子块携带父块的完整内容（用于 LLM 生成时提供更大上下文）
    """
    logger.info(f"开始处理文档: {doc.metadata.get('file_path', 'N/A')}")

    # 根据文件类型选择切分器
    file_extension = os.path.splitext(doc.metadata.get("file_path", ""))[1].lower()
    is_markdown = file_extension == ".md"

    parent_splitter = (
        MarkdownTextSplitter(chunk_size=config.PARENT_CHUNK_SIZE, chunk_overlap=config.CHUNK_OVERLAP)
        if is_markdown
        else RecursiveCharacterTextSplitter(chunk_size=config.PARENT_CHUNK_SIZE, chunk_overlap=config.CHUNK_OVERLAP)
    )
    child_splitter = (
        MarkdownTextSplitter(chunk_size=config.CHILD_CHUNK_SIZE, chunk_overlap=config.CHUNK_OVERLAP)
        if is_markdown
        else RecursiveCharacterTextSplitter(chunk_size=config.CHILD_CHUNK_SIZE, chunk_overlap=config.CHUNK_OVERLAP)
    )

    parent_docs = parent_splitter.split_documents([doc])
    logger.debug(f"文档切分为 {len(parent_docs)} 个父块")

    child_chunks = []
    for j, parent_doc in enumerate(parent_docs):
        parent_id = f"doc_{doc.metadata['hash']}_parent_{j}"
        parent_doc.metadata["parent_id"] = parent_id
        parent_doc.metadata["parent_content"] = parent_doc.page_content
        parent_doc.metadata.update(doc.metadata)

        sub_chunks = child_splitter.split_documents([parent_doc])
        for k, sub_chunk in enumerate(sub_chunks):
            chunk_id = f"{parent_id}_child_{k}"
            sub_chunk.metadata["parent_id"] = parent_id
            sub_chunk.metadata["parent_content"] = parent_doc.page_content
            sub_chunk.metadata["chunk_id"] = chunk_id
            sub_chunk.metadata["id"] = chunk_id  # 用于 Milvus 主键
            sub_chunk.metadata.update(doc.metadata)
            child_chunks.append(sub_chunk)

    logger.info(f"文档共生成 {len(child_chunks)} 个子块")
    return child_chunks
```

> **为什么用父子块策略？**
> - 子块 (400 chars) 用作检索单元：粒度小，语义集中，向量更精准
> - 父块 (1000 chars) 提供上下文：LLM 生成答案时可以看到更大的上下文窗口
> - Markdown 文件用 `MarkdownTextSplitter`：能按标题层级语义切分，效果更好

### Step 5.2 — 编写 `rag/chain.py`

```python
# rag/chain.py
import os
from typing import List
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables import RunnablePassthrough, RunnableConfig
from langchain_core.documents import Document
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI
from loguru import logger
from config import config
from utils.vector_store import VectorStore

# 初始化 LLM（温度稍高，因为需要生成推荐理由）
llm = ChatOpenAI(
    model_name="qwen-plus",
    openai_api_key=config.DASHSCOPE_API_KEY,
    openai_api_base="https://dashscope.aliyuncs.com/compatible-mode/v1",
    temperature=0.1,
)

# 初始化检索器
retriever = VectorStore()

# 查询改写 Prompt
REWRITE_PROMPT = ChatPromptTemplate.from_messages([
    ("system", "你是一位专业的招聘助理，请根据对话历史，将用户的需求改写成一个更具体、更适合用于搜索简历的查询语句。只需要输出改写后的查询，不要有额外解释。"),
    MessagesPlaceholder(variable_name="chat_history"),
    ("user", "{input}"),

# 答案生成 Prompt
ANSWER_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """
作为资深技术招聘官和最终审核人，根据【用人需求】和【简历上下文】推荐最匹配的候选人。

【简历上下文】:
---
{context}
---

输出要求：
1. **最终审核**：作为最后一道关卡，确保简历与【用人需求】高度相关。
2. **精准推荐**：仅当简历高度匹配时，生成推荐理由；忽略勉强相关或无关的简历。
3. **JSON 格式**：严格按以下格式输出可能为空的列表，不添加额外解释：
    ```json
    [
        {{
            "candidate_id": 1,
            "reason": "推荐理由...",
            "file_path": "简历文件名.pdf",
            "doc_hash": "简历哈希值"
        }},
        ...
    ]
    ```
"""),
    ("user", "【用人需求】: {input}"),
])


def _format_docs(docs: List[Document]) -> str:
    """将检索到的文档格式化为 LLM 可读的字符串"""
    if not docs:
        return "未在简历库中找到相关信息。"
    formatted = []
    for doc in docs:
        doc_hash = doc.metadata.get("doc_hash", doc.metadata.get("hash", "N/A"))
        doc_str = (
            f"简历来源文件: {os.path.basename(doc.metadata.get('file_path', 'N/A'))}\n"
            f"简历哈希值: {doc_hash}\n"
            f"内容: {doc.page_content}"
        )
        formatted.append(doc_str)
    return "\n\n---\n\n".join(formatted)


async def get_rag_chain():
    """构建异步 RAG 链"""

    async def retrieve_and_format_context(input_dict: dict, config: RunnableConfig) -> str:
        # 1. 改写查询（让 LLM 优化搜索词）
        rewritten_question = await (REWRITE_PROMPT | llm | StrOutputParser()).ainvoke(
            {"input": input_dict["input"], "chat_history": input_dict["chat_history"]},
            config=config
        )
        logger.info(f"原始查询: '{input_dict['input']}' -> 改写后: '{rewritten_question}'")

        # 2. 异步混合检索（Milvus + ES + Reranker）
        params = input_dict.get("params", {})
        try:
            retrieved_docs = await retriever.aget_relevant_documents(
                query=rewritten_question,
                params=params
            )
        except Exception as e:
            logger.error(f"检索器执行失败: {e}", exc_info=True)
            return "检索简历时发生内部错误，请稍后再试。"

        # 3. 格式化上下文
        context = _format_docs(retrieved_docs)
        logger.debug(f"为LLM准备的上下文: \n{context}")
        return context

    # 构建 LCEL 链
    conversational_rag_chain = (
        RunnablePassthrough.assign(
            context=retrieve_and_format_context
        )
        | ANSWER_PROMPT
        | llm
        | StrOutputParser()
    )

    return conversational_rag_chain
```

> **关键设计点**：
> - `retrieve_and_format_context` 函数中串联了 "查询改写 → 检索 → 格式化" 三步
> - 查询改写让 LLM 将自然语言需求转化为更适合搜索的关键词（如 "找个会AI的" → "人工智能 机器学习 深度学习"）
> - 整个链用 LCEL 的 `|` 管道符串联，可异步调用 `.ainvoke()`

### Step 6.1 — 理解 6 种意图分类

| 意图 | 含义 | 举例 | 路由目标 |
|------|------|------|----------|
| `recruitment` | 新的招聘需求 | "帮我找个前端" | RAG链 |
| `refinement_or_correction` | 修改上一个需求 | "5年经验以上，要男性" | RAG链（有新参数） |
| `follow_up_question` | 追问上次结果 | "第一个候选人会什么?" | 追问链 |
| `general_job_inquiry` | 与招聘相关但不需检索 | "AI产品经理要什么技能?" | 通用问答链 |
| `chit_chat` | 闲聊 | "你好" | 固定回复 |
| `meta_inquiry` | 问系统自身 | "你是谁?" | 固定回复 |
| `fallback` | 无法分类 | "今天天气" | 固定引导回复 |

PRESET_RESPONSES = {
    "chit_chat": "你好！我是您的SmartRecruit智能招聘助手。您可以直接告诉我您的招聘需求，或咨询与招聘相关的通用问题。",

    "fallback": "抱歉，我是SmartRecruit智能招聘助手。您可以尝试告诉我具体的招聘需求，如'我需要一位Java开发工程师'。欢迎随时咨询。"

# 定义链
intent_chain = INTENT_PROMPT | llm
general_qa_chain = GENERAL_QA_PROMPT | llm
parameter_extraction_chain = PARAMETER_EXTRACTION_PROMPT | llm
follow_up_chain = FOLLOW_UP_PROMPT | llm


async def recognize_intent(query: str, chat_history: List[BaseMessage]) -> str:
    """识别用户意图"""
    history_str = "\n".join([f"{msg.type}: {msg.content}" for msg in chat_history])
    try:
        response = await intent_chain.ainvoke({"input": query, "chat_history": history_str})
        cleaned = response.content.strip().removeprefix("```json").removesuffix("```").strip()
        result = json.loads(cleaned)
        intent = result.get("intent", "fallback")
        logger.info(f"意图识别: '{query}' -> '{intent}'")
        return intent
    except Exception as e:
        logger.error(f"意图识别失败: {e}")
        return "fallback"


async def extract_parameters(query: str) -> Dict[str, Any]:
    """从用户需求中提取结构化招聘参数"""
    try:
        response = await parameter_extraction_chain.ainvoke({"input": query})
        cleaned = response.content.strip().removeprefix("```json").removesuffix("```").strip()
        params = json.loads(cleaned)
        logger.info(f"参数提取: '{query}' -> {params}")
        return params
    except Exception as e:
        logger.error(f"参数提取失败: {e}")
        return {"count": 3, "gender": None, "age_min": None, "age_max": None,
                "experience_min": None, "experience_max": None}


class SmartRecruitAgent:
    """主 Agent — 意图识别、路由分发、多轮对话管理"""

    def __init__(self):
        self.rag_chain = None
        self.chat_history: List[BaseMessage] = []

    async def _initialize(self):
        """懒初始化 RAG 链（首次调用时加载）"""
        if not self.rag_chain:
            logger.info("首次初始化RAG链...")
            self.rag_chain = await get_rag_chain()
            logger.info("RAG链初始化完成。")

    async def arun(self, query: str,
                   last_candidates: Optional[List[Dict[str, Any]]] = None) -> Dict[str, Any]:
        """Agent 主入口 —— 异步运行一次对话"""
        await self._initialize()
        intent = await recognize_intent(query, self.chat_history)

        response_content = ""
        returned_candidates = None

        # ---- 路由分发 ----

        # 场景1: 固定回复 (闲聊/元问题/兜底)
        if intent in PRESET_RESPONSES:
            response_content = PRESET_RESPONSES[intent]

        # 场景2: 通用性问题
        elif intent == "general_job_inquiry":
            try:
                response = await general_qa_chain.ainvoke({"input": query})
                response_content = response.content
            except Exception as e:
                logger.error(f"通用问答链失败: {e}")
                response_content = PRESET_RESPONSES["fallback"]

        # 场景3: 追问（需要上一轮的候选人上下文）
        elif intent == "follow_up_question" and last_candidates:
            try:
                context_str = json.dumps(last_candidates, indent=2, ensure_ascii=False)
                response = await follow_up_chain.ainvoke({
                    "input": query,
                    "last_candidates": context_str
                })
                cleaned = response.content.strip().removeprefix("```json").removesuffix("```").strip()
                follow_up_result = json.loads(cleaned)
                response_content = follow_up_result.get("answer", "无法回答此问题。")
                returned_candidates = follow_up_result.get("filtered_candidates", last_candidates)
            except Exception as e:
                logger.error(f"追问链失败: {e}", exc_info=True)
                response_content = "处理您的追问时遇到问题，请重试。"
                returned_candidates = last_candidates

        # 场景4: 招聘需求 / 修正需求（走 RAG 检索）
        elif intent in ["recruitment", "refinement_or_correction", "follow_up_question"]:
            logger.info(f"意图 '{intent}'，走 RAG 检索流程")
            params = await extract_parameters(query)

            try:
                user_only_history = [
                    msg for msg in self.chat_history
                    if isinstance(msg, HumanMessage)
                ]
                rag_input = {
                    "input": query,
                    "chat_history": user_only_history,
                    "params": params
                }
                raw_response = await self.rag_chain.ainvoke(rag_input)

                # 尝试解析为 JSON（候选人列表）
                cleaned = raw_response.strip().removeprefix("```json").removesuffix("```").strip()
                try:
                    returned_candidates = json.loads(cleaned)
                    if isinstance(returned_candidates, list):
                        response_content = "根据您的需求，我为您推荐了以下候选人："
                    else:
                        returned_candidates = None
                        response_content = raw_response
                except json.JSONDecodeError:
                    returned_candidates = None
                    response_content = raw_response

            except Exception as e:
                logger.error(f"RAG链调用失败: {e}", exc_info=True)
                response_content = PRESET_RESPONSES["fallback"]

        else:
            response_content = PRESET_RESPONSES["fallback"]

        # ---- 对话历史管理 ----
        self.chat_history.append(HumanMessage(content=query))
        final_response = {
            "response": response_content,
            "candidates": returned_candidates
        }
        self.chat_history.append(AIMessage(
            content=json.dumps(final_response, ensure_ascii=False)
        ))

        logger.info(f"Agent 最终返回: response={response_content[:100]}..., "
                     f"candidates_count={len(returned_candidates) if returned_candidates else 0}")
        return final_response
```

> **路由设计要点**：
> - `follow_up_question` 有上下文时走追问链（不重新检索），无上下文时降级为 RAG 检索
> - `refinement_or_correction` 重新走 RAG 检索（因为参数变了）
> - 对话历史只保留 `HumanMessage` 传给 RAG 链（避免 AIMessage 的 JSON 干扰改写）
> - 所有结构化输出统一为 `{"response": "...", "candidates": [...]}` 格式

## Phase 8：评估模块 — `eval/evaluator.py`（约 1.5 小时）

### 目标
用 Ragas 框架评估 RAG 系统的检索和生成质量。

### Step 8.1 — 理解评估指标

| 指标 | 含义 |
|------|------|
| `faithfulness` | 答案是否忠实于检索到的上下文（无幻觉） |
| `answer_relevancy` | 答案与问题的相关程度 |
| `context_precision` | 检索到的上下文中相关文档的精确率 |
| `context_recall` | 检索到的上下文中相关文档的召回率 |

### Step 8.2 — 编写 `eval/evaluator.py`

```python
# eval/evaluator.py
import json
import os
import asyncio
from typing import List, Dict, Any, Tuple
from datasets import Dataset
from langchain_openai import ChatOpenAI
from langchain_core.documents import Document
from langchain_core.runnables import RunnableConfig
from langchain_core.output_parsers import StrOutputParser
from loguru import logger
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy, context_recall, context_precision

from config import config
from milvus_model.hybrid import BGEM3EmbeddingFunction
from rag.chain import llm, retriever, REWRITE_PROMPT, ANSWER_PROMPT

logger.add(os.path.join(config.LOG_DIR, "evaluator.log"), rotation="10 MB", encoding="utf-8")

# Ragas 评估专用 LLM
ragas_llm = ChatOpenAI(
    model="qwen-plus",
    api_key=config.DASHSCOPE_API_KEY,
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

# BGE-M3 嵌入适配器（让 Ragas 能调用本地模型）
class RagasBgeM3EmbeddingsAdapter:
    def __init__(self, *args, **kwargs):
        self.bge_function = BGEM3EmbeddingFunction(*args, **kwargs)

    def embed_query(self, text: str) -> List[float]:
        return self.bge_function.encode_queries([text])["dense"][0]

    def embed_documents(self, texts: List[str]) -> List[List[float]]:
        return self.bge_function.encode_documents(texts)["dense"]

ragas_embeddings = RagasBgeM3EmbeddingsAdapter(
    model_name=os.path.join(config.MODEL_PATH, config.EMBEDDING_MODEL),
    device="cpu", use_fp16=False,
)


async def get_rag_response_for_evaluation(
    query: str, params: Dict[str, Any]
) -> Tuple[str, List[Document]]:
    """为评估提供完整的 RAG 响应"""
    rewritten = await (REWRITE_PROMPT | llm | StrOutputParser()).ainvoke(
        {"input": query, "chat_history": []}, config=RunnableConfig()
    )
    logger.info(f"评估查询改写: '{query}' -> '{rewritten}'")

    try:
        retrieved_docs = await retriever.aget_relevant_documents(
            query=rewritten, params=params
        )
    except Exception as e:
        logger.error(f"评估检索失败: {e}")
        return "[]", []

    context_str = "\n".join([doc.page_content for doc in retrieved_docs])
    final_answer = await (ANSWER_PROMPT | llm | StrOutputParser()).ainvoke(
        {"input": query, "context": context_str}, config=RunnableConfig()
    )
    return final_answer, retrieved_docs


def run_evaluation(dataset: Dataset):
    """运行 Ragas 评估"""
    result = evaluate(
        dataset,
        metrics=[faithfulness, answer_relevancy, context_precision, context_recall],
        llm=ragas_llm,
        embeddings=ragas_embeddings,
    )
    return result


def run_sanity_check():
    """健全性检查：用完美数据验证评估工具链正常工作"""
    print("\n--- Part 1: 评估流程健全性检查 ---")
    dataset = Dataset.from_dict({
        "question": ["谁是姚明？"],
        "answer": ["姚明是来自中国的篮球运动员，曾在休斯顿火箭队打球。"],
        "contexts": [["姚明是一名来自中国的篮球运动员，他身高2米26，司职中锋，曾在NBA休斯顿火箭队效力。"]],
        "ground_truth": ["姚明是一名来自中国的篮球运动员，曾在NBA休斯顿火箭队效力。"],
    })
    result = run_evaluation(dataset)
    print(result)
    if result['faithfulness'][0] >= 1.0:
        print("[成功] 健全性检查通过！")
    else:
        print("[警告] 健全性检查未通过，请检查评估工具链。")


async def run_end_to_end_evaluation():
    """端到端评估：用真实 RAG 输出评估系统质量"""
    print("\n--- Part 2: RAG 系统端到端评估 ---")
    test_query = "需要一位熟悉AI大模型的产品经理"
    test_ground_truth = "该候选人拥有AI大模型相关的产品管理经验，具备大语言模型、NLP等技术理解能力。"

    generated_json, retrieved_docs = await get_rag_response_for_evaluation(
        test_query, {"count": 1}
    )

    # 解析推荐理由
    natural_language_answer = ""
    try:
        clean = generated_json.strip().removeprefix("```json").removesuffix("```").strip()
        parsed = json.loads(clean)
        if parsed and isinstance(parsed, list):
            natural_language_answer = parsed[0].get("reason", "")
    except Exception:
        natural_language_answer = ""

    string_contexts = [doc.page_content for doc in retrieved_docs]
    dataset = Dataset.from_dict({
        "question": [test_query],
        "answer": [natural_language_answer],
        "contexts": [string_contexts],
        "ground_truth": [test_ground_truth],
    })

    print(f"问题: {test_query}")
    print(f"RAG 生成理由: {natural_language_answer or '未能解析'}")
    result = run_evaluation(dataset)
    print("\n评估结果:")
    print(result)


async def main():
    run_sanity_check()
    await run_end_to_end_evaluation()

if __name__ == "__main__":
    asyncio.run(main())
```

### Step 9.1 — 编写 `system_data_init.py`

```python
# system_data_init.py
import os
import time
from datetime import datetime
from loguru import logger
import tqdm
from config import config
from utils.document_processor import (
    load_and_hash_document, parse_resume_structure,
    process_document, parser_client
)
from utils.vector_store import VectorStore
from openai import OpenAI
from langchain_core.documents import Document

logger.add(os.path.join(config.LOG_DIR, "system_init.log"),
           rotation="10 MB", encoding="utf-8", level="INFO")

def initialize_system_data():
    """批量导入本地简历"""
    logger.info("=" * 50)
    logger.info("开始执行系统数据初始化...")
    logger.info(f"扫描简历目录: {config.LOCAL_RESUME_DIR}")

    if not os.path.exists(config.LOCAL_RESUME_DIR):
        logger.error(f"简历目录不存在: {config.LOCAL_RESUME_DIR}")
        return

    try:
        vector_store = VectorStore()
        doc_client = OpenAI(
            api_key=config.DASHSCOPE_API_KEY,
            base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
        )
    except Exception as e:
        logger.critical(f"初始化核心组件失败: {e}", exc_info=True)
        return

    supported_extensions = [".md", ".docx", ".pdf", ".txt", ".jpg", ".png"]
    resume_files = [
        f for f in os.listdir(config.LOCAL_RESUME_DIR)
        if os.path.splitext(f)[1].lower() in supported_extensions
    ]

    if not resume_files:
        logger.warning("未找到任何支持的简历文件。")
        return

    logger.info(f"发现 {len(resume_files)} 个简历文件")
    success_count = fail_count = exist_count = 0

    with tqdm.tqdm(total=len(resume_files), desc="处理简历") as pbar:
        for filename in resume_files:
            file_path = os.path.join(config.LOCAL_RESUME_DIR, filename)
            pbar.set_description(f"处理中: {filename}")
            try:
                content, doc_hash = load_and_hash_document(file_path, doc_client)
                structured_data = parse_resume_structure(content, parser_client)
                doc = Document(
                    page_content=content,
                    metadata={
                        "file_path": file_path,
                        "original_name": filename,
                        "hash": doc_hash,
                        "timestamp": datetime.now().isoformat(),
                        **structured_data
                    }
                )
                chunks = process_document(doc)
                success = vector_store.store_resume(doc, chunks, structured_data)
                if success:
                    logger.info(f"成功处理: {filename}")
                    success_count += 1
                else:
                    logger.warning(f"已存在，跳过: {filename}")
                    exist_count += 1
            except Exception as e:
                logger.error(f"处理失败: {filename}, 错误: {e}", exc_info=True)
                fail_count += 1
            pbar.update(1)
            time.sleep(0.1)

    logger.info("=" * 50)
    logger.info("系统数据初始化完成！")
    logger.info(f"成功: {success_count}, 已存在: {exist_count}, 失败: {fail_count}")

if __name__ == "__main__":
    print("即将开始系统数据初始化...")
    print("请确保 Milvus, Elasticsearch, MongoDB 都已启动。")
    initialize_system_data()
    print("初始化流程结束。")
```
